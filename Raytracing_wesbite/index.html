Differences between CPU raytracing and GPU raytracing

Both CPU and GPU ray tracing have the same goal but utilize different computer components. The CPU approach is slower due to its general-purpose design but can handle larger scenes because of its access to system memory. This makes it ideal for non-real-time scenarios. On the other hand, the GPU's specialized design makes it faster and better suited for real-time applications. It's important to note that some modern GPUs have built-in cores specifically for ray tracing, but scenes still need optimization to fit within the GPU's memory limitations.

Information on Denoiser

The denoiser identifies the difference between noise and detail in the image generated by ray tracing. It uses techniques like statistical analysis, which looks at neighboring pixels for areas of high variation and marks them as noise. There are many ways for the denoiser to enhance the image, for example, through bilateral filtering which blurs out high noise areas while maintaining shape edges.

Interactive Path Tracer

Interactive path tracing allows users modify their photo-realistic renders at runtime. The path tracer is implemented with optimization in mind, including an iterative approach via solving the path integral formulation of the light transport problem for increased hardware performance, Russian Roulette for unbiased early path termination, and importance sampling. Supported lighting models are Lambertian surfaces and perfectly specular mirror-like surfaces.

Interactive Camera

The interactive camera supports three unique modes: stationary orbit mode, first person view mode, and creative mode. All camera settings such as FOV, sensitivty, and speed are configurable. For creative mode, there are special settings for controling gravity, jump speed, and the ground height.
